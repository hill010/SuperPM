# Design Brief

## 产品概要
- 产品名称：AI SplitMaster（AI分镜大师）
- 产品类型：Desktop（Electron）
- 目标用户：小白用户（快速上手体验）、AI创作者（创作提效）、短视频创作者（沉淀工作流）
- 核心功能：通过多Agent协作流程（节拍拆解→九宫格→四宫格→动态提示词），将剧本转化为专业分镜提示词和分镜图，每步AI导演审核+人工确认

## 设计方向

### 情绪关键词
- **专业** — 这是干正事的工具，不是玩具。打开就觉得靠谱，信息传达清晰，操作路径明确
- **沉浸** — 中性电影感的界面退到背景，让分镜图和提示词成为视觉焦点。用户长时间使用不疲劳
- **精致** — 细节考究但不过度设计，每个元素都有意义，动画自然流畅让人感觉不到卡顿

### 参考产品

| 参考产品 | 喜欢的方面 | 不喜欢的方面 |
|---------|-----------|-------------|
| Bento Cards SimpleList | 深灰底色层级、白色文字透明度分层、Rubik字体、紧凑卡片布局、整体克制的高级感 | 移动端导向，部分交互偏轻量 |
| Figma | 深色专业工具界面、三栏布局的信息密度、控件位置和交互习惯 | 功能过于复杂，层级太深 |
| Linear | 紧凑但干净的信息密度、流畅自然的过渡动画、控件反馈精准 | 冷色偏重，缺少创作工具的温度 |
| Premiere Pro / DaVinci | 纯中性深灰界面让内容成为焦点、专业工具的信任感 | 信息过载，界面过于密集 |

### 反面参考
- 蓝白管理后台风格：不要 Element UI / Ant Design 那种千篇一律的管理后台
- 花哨动效：不要 Framer 官网那种满屏动效
- 彩色侧边栏：不要 Slack 那种五颜六色的侧边导航

## 视觉规范

### 色彩方向
- **主题模式**：深色为主（必须），浅色模式作为二期可选
- **色彩温度**：中性——不偏冷不偏暖，深灰为底，让内容（分镜图/提示词）自带色彩
- **品牌主色方向**：深灰层级体系（参考 Bento Cards：#121212 底层 / #191919 卡片层 / #282828 悬浮层），极低饱和度的中性灰调
- **强调色方向**：一个低饱和度的强调色用于关键操作和状态标识（如导演审核PASS/FAIL），不抢内容的风头。建议方向：柔和的靛蓝或琥珀色
- **品牌资产**：无现成 logo 或品牌色，从零构建

### 信息密度
- **密度方向**：紧凑
- **参考基准**：接近 Bento Cards SimpleList + Linear 的密度，三栏布局空间利用充分，核心信息一屏可见
- **理由**：产品有三栏结构天然信息分层（导航/工作/预览），紧凑密度让用户不用翻页就能看到关键信息，专业用户高效操作

### 排版方向
- **字体气质**：几何无衬线，干净利落
- **中文字体偏好**：系统默认（Windows 微软雅黑 / Mac 苹方），不额外加载中文字体
- **标题风格**：与正文差异适中，字重区分层级（参考 Bento Cards：500/600/700 三档字重），不需要夸张的标题字号
- **字体参考**：Rubik（英文/数字），系统默认（中文）

### 交互风格
- **动画程度**：适中——每个过渡都自然流畅，但用户不会注意到"有动画"，只会觉得"不卡"
- **过渡效果**：快速渐变（150-250ms ease），页面切换柔和，列表操作流畅，不要弹跳和弹性效果
- **整体节奏**：沉稳但不迟钝，操作即时响应，动画不拖泥带水

## 核心页面视觉备注

### 三栏主工作区
- **核心交互**：左侧项目导航 → 中间工作区 → 右侧预览/审核，三栏联动
- **视觉方向**：高度参考 Bento Cards 的深灰层级体系。左栏最暗（#121212），中栏卡片层（#191919），右栏悬浮层（#282828）或同色。三栏用 1px 低透明度分割线区分，不要粗重边框
- **特殊要求**：三栏比例可拖拽调整，默认 20% / 45% / 35%

### 左侧项目导航
- **核心交互**：项目名+集数列表+阶段步骤条，点击切换
- **视觉方向**：参考 Bento Cards SimpleList 的列表风格——紧凑的分组卡片，hover 时微妙的高亮变化。阶段步骤条用竖向连接线+圆点，当前阶段高亮强调色，已完成打勾
- **特殊要求**：底部 API Key 状态指示灯（绿色圆点=已配置，红色=未配置），小而清晰

### 中间工作区
- **核心交互**：根据当前阶段显示不同内容（剧本输入/配置/生成按钮/修改意见输入）
- **视觉方向**：参考 Bento Cards 的卡片式布局，输入区域用略浅的深灰卡片承载（#282828 背景），主操作按钮用强调色。文本输入框无显式边框，用背景色差区分
- **特殊要求**：生成中状态要有清晰的进度指示（AI思考中/导演审核中），不要模态弹窗遮挡操作

### 右侧预览/审核区
- **核心交互**：Tab切换「产出预览」和「审核意见」，预览区按九宫格/四宫格排版
- **视觉方向**：
  - 九宫格：3×3 网格，每格是一个深色卡片，内含提示词文本。如选择生图则卡片内展示图片+底部提示词
  - 四宫格：2×2 网格，同上风格
  - 审核意见：PASS 用绿色标识，FAIL 用红/橙色标识，审核意见以列表形式展示
- **特殊要求**：九宫格/四宫格的网格卡片之间间距紧凑（8-12px），卡片内有适当内边距。参考 Bento Cards 的卡片间距风格

### 设置页（API Key配置）
- **核心交互**：填写各服务商API Key，验证可用性
- **视觉方向**：简洁表单，参考 Linear 的设置页风格——分组清晰、输入框简洁、验证状态明确
- **特殊要求**：API Key 输入框默认密码遮罩，右侧有显示/隐藏切换

### 项目首页/欢迎页
- **核心交互**：新建项目、打开最近项目
- **视觉方向**：参考 Bento Cards Landing Page 的深色大气的首屏——品牌标识居中、最近项目用卡片网格展示、新建按钮醒目
- **特殊要求**：空状态时（无项目）显示引导文案+新建按钮

## 状态设计
- **空状态**：简短文案引导 + 操作按钮，不用插画。参考 Bento Cards SimpleList 的空列表处理
- **加载状态**：AI生成时用打字机光标动画或脉冲点动画，图像生成用进度条，其他用骨架屏
- **错误状态**：内联错误提示，用强调色的红色变体，不用弹窗。API Key 验证失败在输入框下方显示错误信息

## 补充规范

### 文字层级体系（参考 Bento Cards）
| 层级 | 用途 | 字号 | 字重 | 颜色 |
|------|------|------|------|------|
| 标题 | 页面/面板标题 | 20px | 500 | 白色 95% |
| 正文 | 主要内容文本 | 16px | 400 | 白色 95% |
| 正文强调 | 按钮、标签 | 14-16px | 500 | 白色 95% |
| 辅助文本 | 说明、描述 | 14px | 400 | 白色 70% |
| 标签/徽章 | 状态标签、编号 | 12px | 500-600 | 白色 70% |
| 最小文本 | 时间戳、提示 | 10px | 500-600 | 白色 50% |

### 背景色层级（参考 Bento Cards）
| 层级 | 色值 | 用途 |
|------|------|------|
| 底层 | #121212 | 左侧导航栏、最底层背景 |
| 卡片层 | #191919 | 中间工作区背景、卡片 |
| 悬浮层 | #282828 | 输入框背景、hover状态、右侧面板 |

### 动画规范
| 场景 | 效果 | 时长 |
|------|------|------|
| 页面切换 | 淡入淡出 | 200ms ease |
| 列表项操作 | 平滑位移 | 150ms ease |
| 按钮点击 | 轻微缩放+透明度 | 100ms ease |
| 生成进度 | 脉冲点/打字机光标 | 持续循环 |
| 审核结果出现 | 从下方滑入+淡入 | 200ms ease |
| Tab切换 | 内容淡入淡出 | 150ms ease |
