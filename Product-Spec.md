# Product Spec

## 产品概述

AI分镜Agent桌面端应用——将剧本/梗概通过多Agent协作流程，逐步转化为专业的分镜提示词和分镜图。

**目标用户**：
1. **小白用户**：没有分镜专业背景，想快速把故事变成画面，需要低门槛的一步步引导
2. **AI创作者**：已经在用AI工具做漫画/动画，需要创作提效，核心诉求是工作流标准化和提示词质量
3. **短视频创作者**：需要为拍摄/制作出分镜脚本，核心诉求是沉淀可复用的创作工作流

**核心价值**：用户只需输入剧本，AI自动完成节拍拆解、分镜提示词生成、动态提示词生成，每一步都有AI导演审核+人工确认，确保产出质量。把原本需要专业分镜师数小时的工作压缩到分钟级。

## 应用场景

- **短剧分镜**：短剧编剧小李有一集3分钟的短剧脚本，需要出分镜给摄影师参考。他把脚本贴进来，选择"真人写实+9:16竖屏"，四步走完拿到完整的九宫格分镜提示词和动态提示词，选择生图后直接导出分镜图片给团队。

- **AI漫画创作**：AI创作者小王用Midjourney做国漫风漫画，但每次手写提示词质量不稳定。他输入剧本，选择"国漫+9:16"，系统按方法论生成专业提示词，他复制到Midjourney出图，效率提升3倍。

- **动画前期策划**：动画工作室要向客户提案，策划人员把剧本输入，选择"3D CG+16:9"，30分钟拿到全部分镜图和动态提示词，当天就能开提案会。

- **MV分镜**：独立音乐人要拍MV，把歌词和场景描述输入，选择"写实+16:9"，生成的分镜直接当拍摄参考。

- **教学演示**：小学语文老师想把课文变成连环画，选择"动漫"风格，生成的分镜图直接做课件。

## 功能需求

**核心功能**

- 项目创建：用户点击「新建项目」→ 输入项目名称 → 系统创建项目文件夹结构 → 进入项目工作区
- 剧本输入：用户在文本框输入/粘贴剧本内容 → 系统保存到当前集数 → 作为后续所有阶段的上游输入
- 多集管理：用户添加/切换集数（ep01/ep02/...）→ 每集独立进度追踪 → 各集产物互不干扰
- 视觉配置：用户选择视觉风格（真人写实/3D CG/皮克斯/迪士尼/国漫/日漫/韩漫）、目标媒介（电影/短剧/漫剧/MV/广告）、画幅比例（16:9/9:16）→ 配置写入项目，全流程生效
- AI服务商配置：用户在设置中填入API Key（文本：OpenAI/Claude/通义/文心/DeepSeek等；图像：通义万相/Gemini/DALL-E 3/GPT-image等）→ 系统验证可用性 → 存储到本地配置
- 节拍拆解：用户点击「开始拆解」→ AI分析剧本识别叙事拐点 → 生成节拍拆解表 → AI导演审核 → 展示审核意见 → 用户确认通过/要求修改 → 通过后保存产物
- 九宫格提示词：用户点击「生成Beat Board」→ AI基于节拍拆解生成九宫格提示词 → AI导演审核 → 用户确认 → 通过后保存产物
- 四宫格提示词：用户点击「生成Sequence Board」→ AI基于九宫格生成四宫格提示词 → AI导演审核 → 用户确认 → 通过后保存产物
- 动态提示词：用户点击「生成Motion Prompt」→ AI基于全部分镜生成动态提示词 → AI导演审核 → 用户确认 → 通过后保存产物
- 图像生成（可选）：用户在每个阶段选择是否生图 → 调用已配置的图像API → 展示生成的分镜图 → 图片保存到本地项目目录
- 提示词预览：系统按九宫格/四宫格的排版格式展示提示词 → 用户可直观浏览分镜板 → 支持逐格查看和编辑

**辅助功能**

- 项目进度总览：用户查看当前项目所有集数的完成状态（节拍拆解/九宫格/四宫格/动态提示词各是否完成）
- 产物导出：用户导出当前阶段的产物 → 支持Markdown文档、图片文件、提示词预览版（排版格式）
- 内容修改：用户在任何阶段提出修改意见 → AI根据意见修改 → AI导演重新审核 → 用户确认
- 模板管理：用户可保存当前项目的视觉配置为模板 → 新项目可复用模板快速开始

## UI 布局

### 整体布局
三栏布局，左侧导航区占 20%，中间工作区占 45%，右侧预览/审核区占 35%。

### 左侧 - 项目导航区
- 顶部：项目名称 + 设置入口（齿轮图标）
- 集数列表：显示所有集数（ep01/ep02/...），每集显示完成状态图标，点击切换
- 「添加集数」按钮
- 当前集的阶段导航：竖向步骤条，显示四个阶段（节拍拆解 → 九宫格 → 四宫格 → 动态提示词），当前阶段高亮，已完成阶段打勾，未到达阶段灰显
- 底部：AI服务商配置入口 + API Key状态指示灯（已配置/未配置）

### 中间 - 工作区
- 顶部面包屑：项目名 > 集数 > 当前阶段
- 阶段一「节拍拆解」：
    - 视觉配置区：三个下拉框（视觉风格/目标媒介/画幅比例）
    - 剧本输入区：多行文本框，placeholder「请输入或粘贴剧本内容...」
    - 「开始拆解」主按钮
- 阶段二「九宫格提示词」：
    - 上游产物摘要：节拍拆解表的关键内容折叠展示
    - 「生成Beat Board」主按钮
- 阶段三「四宫格提示词」：
    - 上游产物摘要：九宫格提示词的关键内容折叠展示
    - 「生成Sequence Board」主按钮
- 阶段四「动态提示词」：
    - 上游产物摘要：四宫格提示词的关键内容折叠展示
    - 是否生图开关
    - 「生成Motion Prompt」主按钮
- 通用元素：生成中显示进度状态（AI思考中/导演审核中），修改意见输入框（用户可随时输入修改要求）

### 右侧 - 预览/审核区
- 顶部切换Tab：「产出预览」|「审核意见」
- 产出预览Tab：
    - 节拍拆解：结构化表格展示，可展开每行查看详情
    - 九宫格：3×3网格排版，每格显示提示词文本；如选择生图则每格显示生成的图片+提示词
    - 四宫格：2×2网格排版，每格显示提示词文本；如选择生图则每格显示生成的图片+提示词
    - 动态提示词：列表展示，每条带编号，可展开查看完整内容
- 审核意见Tab：
    - AI导演审核结果（PASS/FAIL）
    - 审核意见详情（问题位置+违反规则+修改方向）
    - 修改记录历史
- 底部操作栏：「确认通过」按钮（通过后解锁下一阶段）|「要求修改」按钮（弹窗输入修改意见）|「导出」按钮

## 用户使用流程

### 首次使用
1. 启动应用 → 进入设置 → 配置AI服务商API Key（文本+图像）→ 验证通过
2. 点击「新建项目」→ 输入项目名称
3. 进入项目 → 默认创建ep01
4. 选择视觉风格、目标媒介、画幅比例
5. 输入/粘贴剧本内容
6. 点击「开始拆解」→ AI生成节拍拆解表 → AI导演审核 → 右侧展示审核结果
7. 用户确认通过 → 解锁九宫格阶段
8. 点击「生成Beat Board」→ AI生成九宫格提示词 → 导演审核 → 用户确认
9. 重复类似流程完成四宫格和动态提示词
10. 任意阶段可选择生图 → 调用图像API → 预览区展示分镜图
11. 全部完成 → 导出产物（Markdown/图片/提示词预览版）

### 迭代修改
1. 用户在任意阶段提出修改意见（输入修改要求）
2. AI根据意见修改 → 导演重新审核 → 用户确认
3. 修改影响下游产物时提示用户：下游阶段需重新生成

### 多集管理
1. 完成一集后点击「添加集数」
2. 输入新集剧本 → 重复工作流
3. 左侧导航切换集数，各集独立进度

## AI 能力需求

| 能力类型 | 用途说明 | 应用位置 |
|---------|---------|---------|
| 文本理解与分析 | 分析剧本结构，识别叙事拐点、角色、场景 | 节拍拆解阶段 |
| 文本生成（分镜提示词） | 基于节拍拆解生成九宫格/四宫格分镜提示词，遵循分镜方法论 | 九宫格、四宫格阶段 |
| 文本生成（动态提示词） | 基于分镜生成图生视频的动态提示词，遵循动态方法论 | 动态提示词阶段 |
| 文本审核 | AI导演按方法论标准审核每个阶段产出，输出PASS/FAIL+修改意见 | 每个阶段生成后 |
| 图像生成 | 根据分镜提示词生成分镜图（可选） | 用户选择生图时 |
| 图像理解 | 分析参考图提取视觉特征（二期：角色/场景参考图上传） | 二期功能 |

## 技术方向

| 维度 | 选择 | 理由 |
|------|------|------|
| 产品类型 | Desktop（Electron） | 用户明确要求桌面端；需要本地文件操作和项目管理 |
| 推荐技术栈 | Electron + React + TypeScript + Tailwind CSS | Electron做桌面壳，React做UI，TS保证类型安全，Tailwind快速样式 |
| 数据存储 | 本地（文件系统 + SQLite/JSON） | 第一版无需服务器，项目数据存本地，API Key加密存本地 |
| 部署方式 | 桌面安装包（Windows/Mac） | Electron打包为原生安装包，用户下载即用 |
| AI接入 | 用户自带API Key（第一版） | 降低第一版复杂度，无需账号体系和计费系统 |

## 技术说明

- **API Key安全**：用户填入的API Key必须加密存储在本地，不明文保存
- **AI服务调用**：通过各服务商官方API调用，需支持SSE流式返回以展示生成进度
- **分镜方法论**：原型的分镜方法论文件（storyboard-methodology-playbook.md、motion-prompt-methodology.md、gemini-image-prompt-guide.md）内嵌到应用中，作为AI生成的系统提示词
- **审核闭环**：AI导演审核FAIL时自动修改并重新提交审核，最多循环3次，超过3次仍FAIL则暂停交由用户处理
- **文件操作**：项目数据存本地文件系统，用户可选择项目存储位置
- **离线可用**：应用本体离线可用，AI功能需要网络连接
