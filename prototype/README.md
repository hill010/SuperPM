# AI SplitMaster - 交互原型

一个用于验证产品交互流程的 HTML 原型，可直接在浏览器中打开体验。

## 使用方法

1. 用浏览器打开 `index.html` 文件
2. 点击各按钮体验页面跳转

## 包含的页面和交互

### 1. 欢迎页 (Welcome)
- 设置入口（右上角齿轮图标）
- 新建项目按钮
- 最近项目列表（点击可进入工作区）

### 2. 设置页 (Settings)
- 返回按钮
- 文本 AI 服务配置
- 图像 AI 服务配置
- API Key 验证状态

### 3. 工作区 (Workspace)
**左侧导航：**
- 项目名称
- 集数列表（支持切换）
- 添加集数按钮
- 四阶段步骤条（可点击切换）
- API 状态指示

**中间工作区（四阶段）：**
- 阶段1「节拍拆解」：视觉配置 + 剧本输入
- 阶段2「九宫格」：上游产物摘要 + 生成按钮
- 阶段3「四宫格」：上游产物摘要 + 生成按钮
- 阶段4「动态提示词」：上游产物摘要 + 生图开关 + 生成按钮

**右侧预览区：**
- 产出预览 / 审核意见 Tab 切换
- 生成结果展示（节拍表/九宫格/四宫格/动态提示词）
- 确认通过 / 要求修改 / 导出 按钮

### 4. 弹窗
- 新建项目弹窗
- 添加集数弹窗
- 修改意见弹窗
- 导出选项弹窗

## 交互流程验证

### 完整流程
1. 欢迎页点击「新建项目」
2. 输入项目名称 → 进入工作区
3. 在节拍拆解阶段输入剧本 → 点击「开始拆解」
4. 等待 AI 生成 → 查看右侧预览结果
5. 点击「确认通过」→ 自动进入九宫格阶段
6. 重复类似流程完成四宫格和动态提示词
7. 任意阶段可点击「导出」选择格式

### 其他操作
- 点击集数列表切换不同集
- 点击「添加集数」创建新集
- 点击左侧步骤条或底部切换器直接跳转阶段
- 点击「要求修改」提交修改意见后重新生成
- 点击右上角设置图标进入设置页

## 技术说明

- 单文件 HTML，无需构建
- 纯 CSS 实现深色主题
- 原生 JavaScript 实现页面切换和交互
- 模拟 AI 生成过程（2.5秒）
