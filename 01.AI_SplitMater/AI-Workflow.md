# AI Workflow

## 目标

AI 工作流负责把用户输入的剧本或创意主题转化为结构化分镜产物，并通过 AI 导演审核与人工确认形成质量闭环。

V1 聚焦「分镜脚本」项目类型。

## Agent 分工

| Agent | 职责 | 主要输入 | 主要输出 |
|---|---|---|---|
| 剧本分析 Agent | 拆解叙事结构、角色、场景、情绪和视觉转折 | 剧本文本、视觉配置 | 节拍拆解表 |
| 分镜生成 Agent | 将节拍表转为九宫格分镜提示词 | 节拍拆解表、视觉配置 | Beat Board |
| 序列扩展 Agent | 将九宫格扩展为四宫格序列分镜 | Beat Board、视觉配置 | Sequence Board |
| 动态提示词 Agent | 为每个分镜生成图生视频提示词 | Sequence Board、视觉配置 | Motion Prompt |
| AI 导演审核 Agent | 按方法论审核阶段产物 | 当前阶段产物、上游产物、审核规则 | PASS/FAIL、问题列表 |
| 修正 Agent | 根据审核意见或用户意见修正产物 | 原产物、问题列表、用户意见 | 修正版产物 |
| 导出整理 Agent | 将结构化产物整理为导出文档 | 项目、集、产物、审核记录 | Markdown、JSON、纯文本 |

## 总流程

```text
用户输入
→ 阶段生成 Agent
→ AI 导演审核 Agent
→ PASS：等待人工确认
→ FAIL：修正 Agent 自动修正
→ 重新审核
→ 最多 3 次自动修正
→ 仍失败则进入手动编辑
```

## 阶段一：节拍拆解

### 输入

- 剧本文本
- 项目类型
- 视觉风格
- 目标媒介
- 画幅比例

### 生成目标

将剧本拆成可服务分镜生成的节拍表，每个节拍必须包含叙事功能和视觉转折。

### 输出结构

```json
{
  "summary": "本集故事摘要",
  "beats": [
    {
      "index": 1,
      "title": "节拍标题",
      "narrativeFunction": "叙事功能",
      "description": "节拍内容",
      "characters": ["角色A"],
      "location": "场景",
      "emotionalTone": "情绪",
      "visualTurn": "视觉转折"
    }
  ],
  "characters": [],
  "locations": []
}
```

### 审核重点

- 是否覆盖剧本主线
- 是否遗漏关键角色或场景
- 节拍是否过粗或过碎
- 每个节拍是否有明确叙事功能
- 是否包含可视觉化的动作、空间或情绪变化

## 阶段二：九宫格 Beat Board

### 输入

- 已确认的节拍拆解表
- 项目视觉配置

### 生成目标

将节拍内容压缩为 9 个关键视觉节点，每个节点形成可用于图像生成的分镜提示词。

### 输出结构

```json
{
  "cells": [
    {
      "index": 1,
      "sourceBeatIds": ["beat-1"],
      "title": "画面标题",
      "imagePrompt": "完整图像提示词",
      "composition": "构图说明",
      "camera": "镜头类型",
      "mood": "情绪氛围"
    }
  ]
}
```

### 审核重点

- 9 格是否覆盖核心剧情起承转合
- 画面之间是否有节奏变化
- 提示词是否具备主体、场景、构图、风格、情绪
- 是否符合项目视觉风格和画幅比例
- 是否出现重复画面或缺乏转折

## 阶段三：四宫格 Sequence Board

### 输入

- 已确认的 Beat Board
- 项目视觉配置

### 生成目标

将每个关键视觉节点扩展为更细的镜头序列，形成 36 个序列分镜提示词。

### 输出结构

```json
{
  "sequences": [
    {
      "index": 1,
      "sourceBeatBoardCellId": "cell-1",
      "shots": [
        {
          "index": 1,
          "imagePrompt": "分镜提示词",
          "shotType": "特写/中景/远景",
          "cameraMovement": "镜头运动",
          "transition": "转场方式"
        }
      ]
    }
  ]
}
```

### 审核重点

- 每组四宫格是否能表达一个清晰动作或情绪推进
- 镜头景别是否有变化
- 镜头顺序是否符合叙事逻辑
- 是否避免同质化构图
- 是否保留上游九宫格的核心意图

## 阶段四：Motion Prompt

### 输入

- 已确认的 Sequence Board
- 项目视觉配置

### 生成目标

为每个分镜生成适合图生视频或动画工具使用的动态提示词。

### 输出结构

```json
{
  "prompts": [
    {
      "index": 1,
      "sourceShotId": "shot-1",
      "prompt": "完整动态提示词",
      "durationSeconds": 4,
      "cameraMovement": "镜头运动",
      "subjectMotion": "主体动作",
      "environmentMotion": "环境运动"
    }
  ]
}
```

### 审核重点

- 是否明确主体动作
- 是否明确镜头运动
- 是否避免运动描述和画面内容冲突
- 是否适合目标媒介节奏
- 是否能被下游视频生成工具理解

## AI 导演审核机制

### 审核输入

- 当前阶段产物
- 上游已确认产物
- 项目视觉配置
- 当前阶段审核规则
- 历史审核记录

### 审核输出

```json
{
  "status": "pass",
  "summary": "审核摘要",
  "issues": [
    {
      "location": "第2拍",
      "rule": "缺乏视觉转折",
      "severity": "medium",
      "description": "问题说明",
      "suggestion": "修改建议"
    }
  ]
}
```

### PASS 标准

- 当前阶段产物结构完整
- 与上游产物一致
- 符合视觉配置
- 具备可执行的提示词质量
- 没有明显叙事断裂或重复

### FAIL 标准

- 缺少关键字段
- 与上游内容冲突
- 视觉描述不可执行
- 重复、空泛或过度抽象
- 镜头/动作/情绪逻辑不清

## 自动修正机制

### 流程

```text
FAIL
→ 修正 Agent 读取问题列表
→ 生成修正版产物
→ AI 导演重新审核
→ 最多重复 3 次
```

### 规则

- 自动修正必须保持原阶段结构不变。
- 自动修正不得删除用户明确保留的内容。
- 自动修正必须优先解决 high severity 问题。
- 每轮修正需要保存为新的 revision。

### 超限处理

3 次自动修正仍 FAIL 后：

- 阶段状态设为 `failed`
- 提示用户手动编辑
- 显示最后一次问题列表
- 用户修改后可点击「继续审核」

## 用户修改意见机制

### 输入

- 用户自然语言修改意见
- 当前阶段产物
- 上游产物
- 历史审核记录

### 输出

- 修正版当前阶段产物
- 新 revision
- 新审核结果

### 下游影响

如果当前阶段已有下游产物：

```text
用户确认修改
→ 系统检测下游阶段
→ 弹窗提示影响范围
→ 用户选择仅修改当前阶段或连锁重新生成
```

选择「仅修改当前阶段」：

- 当前阶段进入重新审核
- 下游阶段暂不改动
- UI 显示下游可能与当前阶段不一致

选择「连锁重新生成」：

- 当前阶段进入重新审核
- 下游阶段标记为 `needs_regeneration`
- 需要按顺序重新生成下游阶段

## 图像生成机制

V1 可先作为可选能力接入。

### 单格生图

- 用户点击某个格子的「生成图片」
- 读取该格 imagePrompt 和视觉配置
- 调用图像 API
- 成功后生成 Asset 并绑定到格子
- 失败后显示错误和重试入口

### 批量生图

- 用户点击「全部生图」
- 系统按格子顺序依次生成或有限并发生成
- 显示整体进度，如「已生成 3/9」
- 失败项保留重试入口

## Prompt 组织建议

每次调用模型建议包含四层信息：

1. System：Agent 角色、任务边界、输出格式约束
2. Methodology：内置分镜方法论和审核标准
3. Context：项目配置、上游产物、历史修正
4. User Request：本次用户输入或修改意见

输出必须优先要求结构化 JSON，UI 文案和导出文档再由应用层转换。
