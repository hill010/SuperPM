## 2026-04-30

### 变更 14：核心工作台多 Agent 协作重构

**变更内容**：
- **布局调整**：工作台从三栏结构（左导航+中操作+右预览）改为双栏结构（左对话+右画布）
- **多 Agent 体系**：引入 6 个专业角色 Agent：
  - 小O（项目管理员）：统筹项目进度和 Agent 调度
  - 编剧：生成剧本（含节拍拆解）、支持对话修改
  - 角色设计师：生成角色卡片（主视觉图+三视图+文字描述）
  - 场景设计师：生成场景设定（主图+描述+多视图四宫格）
  - 分镜师：生成宫格图分镜（四宫格/九宫格）+ Motion Prompt
  - 音效总监：生成音效方案（背景音乐+音效描述）
- **交互模式**：对话驱动 + 按钮触发，Agent 主动推进流程，用户通过自然语言交互
- **核心工作流更新**：
  - 旧：节拍拆解 → 九宫格 Beat Board → 四宫格 Sequence Board → 动态提示词
  - 新：编剧 → 角色设计 → 场景设计 → 分镜 → 音效
- **新增独立阶段**：
  - 角色设计：产出角色卡片（主视图+三视图+描述），支持资产库复用
  - 场景设计：产出场景设定（主图+描述+多视图四宫格），支持资产库复用
- **审核机制调整**：从 AI 导演统一审核改为各 Agent 自审 + 人工确认
- **画布功能增强**：支持在画布上直接编辑产出（调整分镜顺序、编辑描述、裁剪等）
- **宫格图模式**：分镜阶段改为 OiiOii 那种宫格图模式，分镜师按复杂度自动生成四宫格/九宫格
- **资产复用**：角色和场景资产可加入资产库，支持跨项目调用

**变更位置**：Product-Spec.md（核心工作流、工作台页面定义、项目类型、功能优先级）

**变更原因**：参考 OiiOii.ai 平台设计，将线性按钮驱动流程改为多 Agent 对话驱动模式，提升用户体验和创作效率

### 变更 13：产品规格文档拆分

**变更内容**：
- 将 `Product-Spec.md` 收敛为产品总纲、页面定义、功能优先级和技术方向
- 新增 `MVP-Scope.md`，明确 V1 必做、可选、不做范围和验收路径
- 新增 `Data-Model.md`，定义本地目录结构、核心实体、状态机和导出结构
- 新增 `AI-Workflow.md`，定义 Agent 分工、四阶段输入输出、审核和修正机制
- 明确 V1 优先采用自带 API + 本地存储，账号/充值/平台算力延后到 V2

**变更位置**：Product-Spec.md、MVP-Scope.md、Data-Model.md、AI-Workflow.md

**变更原因**：原规格文档过大，产品范围、技术实现和 AI 流程混在一起，不利于后续实现和评审
