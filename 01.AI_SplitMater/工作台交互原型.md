# 工作台交互原型

> 版本：v1.0 | 日期：2026-04-30 | 基于产品规格定义三栏布局与四阶段工作流

---

## 一、页面定位

**核心任务**：完成分镜脚本的四个阶段（节拍拆解 → 九宫格分镜 → 四宫格序列 → 动态提示词），每阶段通过 AI 导演审核后人工确认。

**布局原则**：
- 三栏固定结构，左窄右宽
- 阶段锁定机制：前一阶段未确认，后一阶段灰显不可操作
- 右栏实时展示产出和审核结果

---

### 3.2 阶段状态图标

| 状态 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| 已确认 | ✓ | 绿色 | 阶段完成，可进入下一阶段 |
| 进行中 | ● | 蓝色 | 当前正在操作或审核中 |
| 锁定 | ○ | 灰色 | 前置阶段未完成 |
| 审核失败 | ✗ | 红色 | AI 审核未通过 |
| 等待生成 | ◌ | 黄色 | 阶段已解锁，等待点击生成 |

## 六、AI 审核与修正流程

### 6.1 审核流程图

```
[点击生成]
    ↓
[Loading 状态]
    ↓
[AI 生成产物]
    ↓
[AI 导演自动审核]
    ↓
    ├── PASS → 显示通过状态 → 用户确认 → 解锁下一阶段
    │
    └── FAIL → 显示问题列表 → 用户选择：
                    │
                    ├── 自动修正 → 重新生成 → 重新审核
                    │       ↓
                    │   （循环最多 3 次）
                    │       ↓
                    │   3 次后仍 FAIL → 强制手动编辑
                    │
                    └── 手动编辑 → 用户修改内容 → 提交审核
```

### 6.2 修正次数计数规则

| 场景 | 次数变化 | 说明 |
|------|----------|------|
| 点击「自动修正」 | +1 | 消耗一次自动修正机会 |
| 点击「重新生成」（输入修改意见后） | +1 | 消耗一次 |
| 手动编辑后提交 | 不计入 | 不消耗自动修正次数 |
| 达到 3 次上限 | 禁用自动修正 | 只能手动编辑 |

## 八、关键交互规则

### 8.1 阶段切换规则

| 当前状态 | 允许操作 | 禁止操作 |
|----------|----------|----------|
| 阶段未解锁 | 无 | 所有生成操作 |
| 阶段进行中 | 取消生成 | 切换到其他阶段 |
| 阶段审核中 | 确认/修改/导出 | 生成下一阶段 |
| 阶段已确认 | 查看产物 | 重新生成（需确认影响） |

### 8.2 按钮状态

| 按钮 | 启用条件 | 禁用条件 |
|------|----------|----------|
| 开始拆解/生成 | API 已配置，阶段已解锁 | API 未配置，阶段锁定 |
| 确认通过 | 审核状态 = PASS | 审核状态 = FAIL 或 生成中 |
| 要求修改 | 审核状态 = FAIL | 审核状态 = PASS |
| 自动修正 | 修正次数 < 3 | 修正次数 ≥ 3 |
| 导出 | 有产物 | 无产物 |

### 8.3 自动保存规则

| 触发时机 | 保存内容 |
|----------|----------|
| 输入文本时（防抖 2s） | 用户输入 |
| 生成完成时 | 全部产物 |
| 审核完成时 | 审核结果 |
| 确认通过时 | 阶段状态 |
| 切换阶段/集数时 | 当前状态 |

## 九、动画与微交互

### 9.1 进入动画

- 阶段步骤条：当前阶段高亮从左向右滑入
- 产物预览：内容从上方淡入 + 轻微上移

### 9.2 状态切换动画

- PASS 状态：绿色对勾从中心放大出现，伴随轻微缩放弹性
- FAIL 状态：红色叉号抖动一次，问题列表依次滑入
- 生成进度：进度条平滑增长，数字跳动

### 9.3 按钮交互

- 主操作按钮：悬停时轻微上移 + 阴影加深
- 确认通过按钮：点击后绿色光晕扩散
- 禁用按钮：半透明 + 禁止光标
