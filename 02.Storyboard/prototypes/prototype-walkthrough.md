## 原型文件

- `prototypes/storyboard-web-prototype.html`

## 交互复查

### 首次使用路径

1. 打开 HTML 后进入登录页。
2. 点击“登录并领取试用积分”进入项目列表。
3. 项目列表可看到积分余额、任务队列、新建项目入口、空状态模拟和已有项目卡片。
4. 点击任一项目卡片进入工作台。

结果：路径符合 Product Spec 的首次使用流程。登录页没有做营销长页，符合 Design Brief。

### A 闭环：文本到分镜表

1. 工作台左侧展示脚本文本和预计拆镜消耗。
2. 点击“生成分镜”后打开任务队列。
3. 任务队列显示文本拆镜任务、冻结积分、进度、完成后扣除积分。
4. 中间分镜列表展示镜头编号、时长、景别、核心内容、动作、首尾帧状态。
5. 镜头更多菜单可复制、插入、移动和删除，并自动重编号。
6. 点击“导出”可选择 Markdown/CSV，并配置是否包含图片和提示词。

结果：A 闭环已经具备完整原型交互。

### B 闭环：文本到首尾帧图

1. 在镜头卡片或右侧镜头详情中点击“首帧”或“尾帧”。
2. 系统显示预计消耗 8 积分并创建生成任务。
3. 对应缩略图进入加载状态。
4. 任务成功后缩略图更新为已生成状态，素材库出现资产。
5. 尾帧失败样例展示“失败 · 已退回”任务和失败镜头状态。
6. 批量生成会先进入积分不足弹层，显示预计消耗、当前余额、购买积分包和升级订阅入口。

结果：B 闭环的任务、扣费、失败退回、素材绑定和余额不足分支都能在原型中看到。

### 时间线草图

1. 点击工作台顶部“时间线草图”标签。
2. 原型展示轻量横向镜头时间线。
3. 页面明确说明第一版只展示镜头顺序和时长，不做视频剪辑。

结果：时间线标签不再是空入口，并且不会误导用户以为第一版包含完整剪辑能力。

### 账户与权限

1. 点击右上角头像打开账户菜单。
2. 菜单包含个人资料、订阅积分、账单记录、无权限页模拟和退出登录。
3. 点击“模拟无权限页”进入 403/Not Found 状态。

结果：账户入口、权限错误状态和第一版个人项目边界已补齐。

## 一致性检查

### 与 Product Spec 一致

- 第一版明确是 Web SaaS，不是桌面复刻。
- 第一版只覆盖 A+B，不出现视频生成和 TTS。
- 权限模型表现为个人工作区，不出现团队邀请入口。
- 计费模型表现为免费试用、订阅、积分包。
- 所有 AI 长任务都通过任务队列表达，没有阻塞式等待。
- 图片资产与镜头绑定，并可进入素材库。
- 失败任务退回积分、余额不足拦截和导出配置都有 UI 表达。

### 与 Design Brief 一致

- 默认深色优先。
- 三栏工作台结构成立。
- 信息密度适中偏紧凑。
- 卡片只用于项目、镜头、素材、任务，没有把大页面 section 做成装饰卡片。
- 任务状态和积分消耗透明。
- 登录页克制，没有营销型长页。
- 时间线保持轻量，不抢第一版核心工作流。

## 仍需正式开发定稿

这些不再是原型缺口，而是正式产品实现前需要定稿的细节：

1. 镜头重排方式  
原型使用上移/下移按钮模拟。正式产品可以继续保留按钮，也可以加拖拽排序。拖拽体验更自然，但实现和移动端适配成本更高。

2. 批量生成的真实扣费规则  
原型展示余额不足分支。正式实现要明确：创建任务时冻结积分，单张成功后扣除，失败释放冻结积分。

3. 时间线第一版范围  
原型把时间线定位为“草图”。正式开发时如果资源紧张，可以保留只读时间线；不要在第一版做剪辑级时间线。

4. 账户菜单具体页面  
原型只展示菜单和计费弹层。正式开发需要个人资料页、账单记录页和退出登录逻辑。

5. 项目空状态触发方式  
原型用按钮模拟空状态。正式系统应由项目数量自动决定。

## 建议

当前 HTML 原型已经足够支撑下一步 `dev-planner`：它明确了第一版页面、主要状态、交互入口和边界。下一步开发计划应把“登录/项目列表/工作台骨架/任务队列/计费模型”作为基础 Phase，把图像生成接入放在后续 Phase。
