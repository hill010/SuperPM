# Product Spec: Storyboard Web

## 产品概述

Storyboard Web 是一个面向外部创作者的 AI 分镜创作 SaaS。它帮助短视频创作者、广告创意团队、独立导演、动画前期策划和内容工作室，把文字脚本快速转成可编辑的分镜列表，并进一步生成首帧/尾帧视觉草图，形成可导出、可复用、可继续生成视频的前期创作资产。

目标用户不是纯技术用户，而是有创意内容生产需求、但不想手工整理镜头表和提示词的人。他们的核心痛点是：脚本到分镜的拆解耗时，镜头提示词难写，视觉参考难统一，给团队或客户沟通时缺少清晰的前期材料。

第一版核心价值是：用户登录后创建项目，输入脚本，让 AI 自动拆解分镜，用户在网页工作台中编辑镜头信息，生成每个镜头的首帧和尾帧图，并导出分镜表或图文资料。第二版再接入视频生成、TTS、最终合成和剪辑导出。

## 应用场景

- 短视频创作者准备剧情短片：创作者输入一段 1-3 分钟短片脚本，系统拆出镜头列表、景别、动作、场景和首尾帧提示词，创作者快速修改后导出分镜表给摄影师。
- 广告创意团队做提案：策划把客户 brief 和创意脚本输入系统，生成带首尾帧图的 storyboard，用于内部评审或客户沟通。
- 动画前期策划出草案：动画导演输入故事段落，先生成分镜结构和视觉草图，再交给美术团队精修。
- 独立导演整理镜头语言：导演把剧本文字转成镜头卡片，调整时长、景别、运镜、场景说明，形成拍摄前的镜头计划。
- 内容工作室复用项目资产：团队成员为同一类账号或系列内容沉淀角色、场景、风格和项目模板，减少重复提示词编写。

## 功能需求

### 核心功能

- 用户注册与登录：用户使用邮箱、第三方登录或账号密码注册登录，系统创建个人工作区，用户只能访问自己的项目和资产。
- 个人工作区：用户进入系统后查看自己的项目列表、最近编辑项目、额度余额和订阅状态。
- 项目创建：用户创建项目并填写项目名称、片长目标、画幅比例、创作目标、目标受众、视频基调和核心信息，系统保存为项目基础信息。
- 脚本输入与 AI 拆镜：用户输入脚本文本并选择期望镜头数量，点击“生成分镜”，系统调用文本 AI 拆解出镜头列表，包括镜头编号、时长、景别、核心内容、动作指令、场景设定、首帧提示词、尾帧提示词、视频提示词草稿。
- 分镜列表编辑：用户在工作台中新增、删除、复制、重排镜头，系统实时保存镜头顺序和镜头字段。
- 镜头详情编辑：用户选中镜头后编辑景别、核心内容、动作、场景、首帧提示词、尾帧提示词、负面提示词、构图、光线、时间段、色调、镜头类型和时长。
- 首帧/尾帧图生成：用户对单个镜头点击“生成首帧”或“生成尾帧”，系统消耗积分创建异步生成任务，完成后把图片保存到云存储并绑定到镜头。
- 批量图片生成：用户选择多个镜头后批量生成首帧、尾帧或两者，系统排队执行并展示每个任务状态。
- 素材库：用户查看项目内生成的图片、上传的参考图和镜头资产，系统按项目、镜头和资产类型归档。
- 导出分镜表：用户导出 Markdown、PDF 或 CSV 格式的分镜资料，内容包括镜头编号、画面图、核心内容、动作、场景、时长和提示词。
- AI 任务队列：用户查看文本拆镜、图片生成、批量生成任务的状态，系统显示排队中、执行中、成功、失败和扣费情况。
- 额度与计费：系统为新用户提供免费试用额度，订阅用户每月获得固定积分，用户可购买额外积分包，图片生成和未来视频/TTS 按积分扣费。

### 辅助功能

- API Key 与模型配置：第一版默认使用平台统一 AI Provider，不把用户 API Key 暴露给前端；后续可开放自带 Key 模式。
- 项目搜索与筛选：用户按项目名称、更新时间、生成状态搜索项目。
- 自动保存：用户编辑项目、镜头和提示词时系统自动保存，避免刷新丢失。
- 失败重试：图片生成失败时系统记录失败原因，未产生成本的失败任务退还积分或不扣积分。
- 账户设置：用户查看个人资料、订阅状态、积分余额、账单记录和退出登录。
- 团队结构预留：数据库预留 workspace/team 字段，第一版 UI 不开放团队邀请和共享。

### 第二版功能

- 视频生成：根据镜头的视频提示词、首尾帧参考和参数生成短视频片段。
- TTS 配音：根据镜头台词或旁白生成音频。
- 最终合成：把已生成视频和音频按镜头顺序合成为粗剪视频。
- 剪辑导出：导出剪映草稿或其他剪辑工程格式。
- 团队协作：开放团队空间、成员邀请、项目共享、角色权限。

## UI 布局

### 整体布局

产品采用工作台式 Web 应用，不做营销页作为登录后的第一屏。登录后进入项目列表；打开项目后进入三栏工作台：

- 左侧栏：项目导航、项目基础信息、素材库入口、生成任务入口。
- 中间主区：分镜列表或轻量时间线视图，展示镜头卡片、首尾帧预览、状态和批量操作。
- 右侧栏：当前选中镜头的详细编辑器，包含文本字段、图像生成参数和生成按钮。
- 顶部栏：产品名、当前项目名、积分余额、导出、账户菜单。
- 底部或抽屉：任务队列，显示异步生成进度和失败信息。

### 项目列表页

- 顶部显示产品名、创建项目按钮、积分余额和账户菜单。
- 主区显示项目卡片列表，卡片包含项目名、更新时间、镜头数量、已生成图片数量和最近缩略图。
- 空状态显示创建项目入口，不使用营销式大段介绍。

### 项目工作台

- 左侧栏固定宽度，包含项目信息、素材库、生成记录和导出入口。
- 中间主区顶部有视图切换：分镜列表、时间线草图。
- 分镜列表使用纵向镜头卡片，每张卡片显示镜头编号、时长、景别、核心内容、首帧图、尾帧图和生成状态。
- 右侧编辑器按分组展示：基础信息、首帧、尾帧、视频提示词草稿、生成参数。
- 任务队列以可折叠抽屉呈现，不遮挡主要编辑区。

## 用户使用流程

### 首次使用

1. 用户注册或登录。
2. 系统创建个人工作区并发放免费试用积分。
3. 用户点击“新建项目”。
4. 用户填写项目名称、画幅比例、目标片长和创作目标。
5. 用户进入项目工作台。

### A 闭环：文本到分镜表

1. 用户粘贴脚本或故事梗概。
2. 用户选择期望镜头数量或使用自动拆分。
3. 用户点击“生成分镜”。
4. 系统创建文本拆镜任务并调用 AI。
5. 系统返回镜头列表并写入项目。
6. 用户编辑镜头字段、调整顺序和时长。
7. 用户导出 Markdown、PDF 或 CSV 分镜表。

### B 闭环：文本到首尾帧图

1. 用户完成或编辑分镜列表。
2. 用户选择单个镜头或批量选择多个镜头。
3. 用户点击“生成首帧”“生成尾帧”或“生成首尾帧”。
4. 系统检查积分余额，余额不足时引导购买积分或升级订阅。
5. 系统创建图片生成任务，任务完成后保存图片到云存储。
6. 用户在镜头卡片和素材库中查看生成图片。
7. 用户导出带图片的分镜资料。

### 计费流程

1. 新用户获得免费试用积分。
2. 文本拆镜消耗少量积分或在免费额度内限制次数。
3. 图片生成按张扣除积分。
4. 用户积分不足时，系统提示升级订阅或购买积分包。
5. 用户完成支付后，系统增加订阅权益或积分余额。

## AI 能力需求

| 能力类型 | 用途说明 | 应用位置 |
| --- | --- | --- |
| 文本理解与结构化生成 | 分析脚本，拆解镜头，生成镜头字段、提示词和参数 | 点击“生成分镜”时 |
| 提示词优化 | 根据镜头核心内容、动作、场景和风格生成更稳定的首尾帧提示词 | 镜头详情编辑器和 AI 拆镜结果生成时 |
| 图像生成 | 根据首帧/尾帧提示词生成分镜图或视觉草图 | 单镜头或批量图片生成时 |
| 图像理解 | 后续用于分析用户上传的角色、场景参考图，提取视觉特征 | 第一版可预留，第二版增强 |
| 安全审核 | 检查用户输入和生成请求是否违反平台内容规则 | 文本拆镜和图片生成前 |

## 技术方向

| 维度 | 选择 | 理由 |
| --- | --- | --- |
| 产品类型 | Web SaaS | 外部创作者需要登录、多用户、云存储、权限、计费和跨设备访问 |
| 前端 | Next.js + TypeScript + Tailwind CSS | 适合复杂工作台、路由、组件化和后续部署；前端体验比桌面 UI 迁移更灵活 |
| 后端 | ASP.NET Core Web API | 现有桌面项目的 Domain/Application/Infrastructure 是 C# 资产，复用成本最低 |
| 数据库 | PostgreSQL | 适合 SaaS 多用户、计费、任务、资产和未来团队协作；开发期可用 SQLite 做局部验证 |
| 文件存储 | S3 兼容对象存储或云厂商对象存储 | 图片、视频、上传素材不能继续依赖本地路径 |
| 异步任务 | 后端任务队列 + Worker | AI 图片生成、未来视频/TTS 都是长任务，不能阻塞 HTTP 请求 |
| 实时状态 | SignalR 或 WebSocket | 前端需要看到任务进度、失败原因和生成完成状态 |
| 支付 | Stripe 或国内支付渠道抽象层 | 先抽象 Billing Service，具体渠道按上线地区决定 |
| 部署 | Web 前端托管 + API/Worker 服务 + 托管数据库 + 对象存储 | 支持外部用户访问、扩容和独立运维 |

## 技术说明

- 现有 `02.Storyboard` 的 `Domain/Entities`、`Application/Services`、`Infrastructure/Services`、`Prompts/*.json` 是可复用资产。
- 现有 Avalonia 的 `App/Views`、`App/ViewModels`、`App/Controls` 不直接迁移，Web 版用 React 组件重做。
- 桌面端本地文件路径要改造成对象存储资源模型，数据库保存 asset id、storage key、public/private URL 和元数据。
- 第一版所有生成任务都要走后端，前端不能直接持有平台 AI Key。
- 计费扣费必须和任务状态绑定：创建任务前预扣或冻结积分，失败后释放或退回。
- 权限第一版只开放个人账户；数据库预留 `Workspace`、`WorkspaceMember`、`Project.OwnerUserId`、`Project.WorkspaceId`。

## 数据模型草案

| 表名 | 用途 |
| --- | --- |
| Users | 保存登录用户、邮箱、头像、注册时间和状态 |
| Workspaces | 保存个人工作区，预留团队空间 |
| WorkspaceMembers | 保存用户与工作区关系，第一版默认每个用户一个 owner 成员 |
| Projects | 保存项目基础信息、创作目标、画幅、片长、用户归属 |
| Shots | 保存镜头编号、时长、核心内容、动作、场景、提示词和生成参数 |
| Assets | 保存上传素材、生成图片、未来生成视频/音频的存储信息 |
| GenerationJobs | 保存 AI 任务类型、状态、输入、输出、错误、积分扣费记录 |
| CreditAccounts | 保存用户或工作区积分余额 |
| CreditTransactions | 保存积分发放、冻结、扣除、退回和购买记录 |
| Subscriptions | 保存订阅计划、状态、周期、权益和支付渠道信息 |
| BillingEvents | 保存支付回调、发票、退款和订阅变更事件 |

## 权限规则

| 操作 | 第一版规则 |
| --- | --- |
| 查看项目 | 只有项目 owner 可以查看 |
| 编辑项目 | 只有项目 owner 可以编辑 |
| 删除项目 | 只有项目 owner 可以删除 |
| 查看资产 | 只有资产所属项目 owner 可以查看 |
| 创建生成任务 | 用户必须登录且拥有项目编辑权限 |
| 扣除积分 | 从用户个人工作区积分账户扣除 |
| 团队协作 | 第一版不开放 UI，仅预留数据结构 |

## 计费规则

| 项目 | 第一版规则 |
| --- | --- |
| 免费试用 | 新用户注册后获得少量积分，用于体验文本拆镜和少量图片生成 |
| 订阅 | 订阅用户每月获得固定积分和更高并发/更高导出限制 |
| 积分包 | 用户可购买额外积分，积分用于图片生成和后续视频/TTS |
| 文本拆镜 | 可免费限次或低积分消耗 |
| 图片生成 | 按生成张数扣除积分 |
| 失败任务 | AI 服务未成功返回可用结果时退还冻结积分 |
| 第二版视频/TTS | 按生成秒数、模型成本和分辨率扣除积分 |

## 第一版范围

第一版必须完成：

- 登录、个人账户、个人工作区。
- 项目列表和项目工作台。
- 文本输入到 AI 分镜列表。
- 分镜列表编辑和镜头详情编辑。
- 单张和批量首尾帧图生成。
- 云存储保存图片资产。
- 任务队列和生成状态。
- 免费试用、订阅、积分包的数据结构和基础扣费逻辑。
- 导出 Markdown、CSV，PDF 可作为增强项。

第一版明确不做：

- 视频生成。
- TTS 配音。
- 最终视频合成。
- 剪映草稿导出。
- 团队邀请和共享项目 UI。
- 用户自带 AI Key。
